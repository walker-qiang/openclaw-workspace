# 数据真实性自查功能

## ✅ 已实现

### 1. 数据验证模块 (`scripts/utils/data_validator.py`)

**验证维度**:

#### 行情数据验证
- ✅ 股价范围检查 (0.01 - 100 万)
- ✅ PE 合理性检查 (-1000 - 1000)
- ✅ PB 合理性检查 (0.01 - 100)
- ✅ 涨跌幅异常检测 (-50% - 50%)
- ✅ 市值范围检查
- ✅ 52 周高低点一致性检查

#### 财务数据验证
- ✅ ROE 合理性检查 (-100% - 200%)
- ✅ 负债率检查 (0% - 100%)
- ✅ 毛利率异常检测
- ✅ 增长率异常检测

#### 交叉验证
- ✅ EPS 数据一致性（行情 vs 财务）
- ✅ BVPS 数据一致性（行情 vs 财务）

### 2. 可信度评分

根据验证结果计算 0-100 分可信度：

| 分数 | 等级 | 图标 | 说明 |
|------|------|------|------|
| 90-100 | 高 | 🟢 | 数据可靠 |
| 70-89 | 中 | 🟡 | 数据基本可靠 |
| 0-69 | 低 | 🔴 | 数据可能有问题 |

### 3. 报告展示

在分析报告中添加"数据可信度"章节：

```markdown
### 🔍 数据可信度：🟢 100.0% (高)

数据验证通过，无明显问题
```

或

```markdown

**数据问题**:
- PE 异常：95.5 (合理范围：-1000-1000)
- ROE 过高 (17254%)，可能是数据错误
```

## 📊 修复的问题

### 问题 1：美股 PE 数据错误

**原因**: 字段映射错误

**修复**: 
- 改用腾讯美股 API
- 正确字段映射：PE=字段 39, PB=字段 64

**修复前**: PE 显示 95.52（实际是价格）  
**修复后**: PE 显示 41.71（正确）

### 问题 2：A 股 PE 字段错误

**原因**: 使用了错误的字段索引

**修复**:
- 动态 PE (TTM) = 字段 40
- 静态 PE = 字段 39
- 优先使用动态 PE

## 🎯 验证规则详情

### 股价验证
```python
REASONABLE_RANGES = {
    'price': (0.01, 1000000),  # 0.01 元 - 100 万元
}
```

### PE 验证
```python
REASONABLE_RANGES = {
    'pe': (-1000, 1000),  # 亏损到 1000 倍
}
```

特殊处理：
- PE < 0: 标记为"公司处于亏损状态"
- PE > 35: 标记为"估值偏高"
- PE > 100: 标记为"严重高估"

### ROE 验证
```python
REASONABLE_RANGES = {
    'roe': (-100, 200),  # -100% 到 200%
}
```

特殊处理：
- ROE > 100%: 标记为"可能是数据错误"
- ROE > 50%: 标记为"异常高"
